# Contributing to wos_audit

Thank you for your interest in contributing to `wos_audit`!

## Code of Conduct

Please be respectful and professional in all your interactions with the project.

## How to Contribute

1.  **Fork the repository** (if you are an external contributor).
2.  **Create a new branch** for your feature or bug fix:
    ```bash
    git checkout -b feature/your-feature-name
    ```
3.  **Make your changes**.
4.  **Ensure tests pass**:
    ```bash
    pytest
    ```
5.  **Commit your changes** with a clear and descriptive commit message.
6.  **Push your branch** to your fork or the main repository.
7.  **Submit a Pull Request**.

## Coding Standards

- Follow [PEP 8](https://www.python.org/dev/peps/pep-0008/) for Python code style.
- Ensure all new features are accompanied by tests.
- Keep functions and classes focused and well-documented.

## Pull Request Guidelines

- Provide a clear title and description for your PR.
- Reference any related issues.
- Keep PRs focused on a single change.
- Be prepared to address feedback during the review process.
